## üìã Project Overview

### **Problem Statement**
UTM bus users experience frustration due to:
- **Unpredictable waiting times** (90.9% of users)
- **Lack of real-time information** (72.7% of users)
- **Difficulty navigating routes** (72.7% of users)
- **Overcrowding during peak hours** (45.5% of users)

### **Our Solution**
A Modern UTM Bus Navigation System featuring:
- **Real-time GPS tracking** of all campus buses
- **Accurate arrival predictions** with countdown timers
- **Interactive route maps** with color-coded systems
- **Personalized notifications** for delays and arrivals
- **Accessibility features** for diverse user needs

### **Key Statistics**
- **22 users surveyed** across students, staff, and visitors
- **100% demand** for real-time tracking features
- **90% task completion rate** during usability testing
- **85% reduction** in user-reported waiting anxiety

## üéØ Design Thinking Process

### **1. Empathy Phase**
- Conducted interviews with 13 users at bus stops
- Created personas: **Ali** (Student) and **Mr. Lee** (Visitor)
- Observed user behavior during peak hours
- Documented emotional pain points and frustrations

### **2. Define Phase**
- Synthesized research using affinity mapping
- Identified 5 core problem areas
- Developed "How Might We" questions for ideation
- Created clear problem statements for each user segment

### **3. Ideate Phase**
- Brainstormed 15+ potential solutions
- Evaluated ideas based on feasibility and impact
- Selected top 3 solutions through team voting
- Used AI assistance for idea expansion and refinement

### **4. Prototype Phase**
- Developed 3 iterative versions in Figma
- Added features based on user feedback
- Created high-fidelity interactive prototype
- Implemented accessibility and customization options

### **5. Test Phase**
- Conducted usability tests at Center Point bus stop
- Collected feedback through observation and surveys
- Iterated designs based on user suggestions
- Validated solution effectiveness with target users

## üõ†Ô∏è Technical Features

### **Core Functionality**
- **Real-time Tracking:** Live GPS location of all campus buses
- **Arrival Predictions:** Accurate ETAs using historical data
- **Route Planning:** Interactive maps with best route suggestions
- **Notifications:** Proactive alerts for delays and arrivals
- **Accessibility:** Dark mode, text scaling, voice guidance

### **User Experience**
- **Simple Login:** UTM ID or guest access
- **Intuitive Interface:** Clean design with clear navigation
- **Personalization:** Favorite routes and frequent stops
- **Multi-language:** Support for Malay, English, Mandarin, Tamil
- **Offline Access:** Basic functionality without internet

### **System Architecture**
- **Frontend:** Mobile-responsive web application
- **Backend:** Cloud-based server for real-time data
- **Database:** User preferences and historical patterns
- **Integration:** UTM authentication system
- **APIs:** GPS tracking, mapping, notification services

## üìä Research Findings

### **User Demographics**
- **Students:** 18.2% (Regular commuters with class schedules)
- **Academic Staff:** 63.6% (Primary users with fixed routines)
- **Visitors:** 18.2% (Occasional users needing navigation help)

### **Top Pain Points**
1. **Long waiting times** without information (90.9%)
2. **Uncertain arrival times** causing anxiety (72.7%)
3. **Difficulty understanding** complex routes (72.7%)
4. **Overcrowding** during peak hours (45.5%)
5. **Lack of information** for visitors (27.3%)

### **Most Requested Features**
1. **Real-time bus tracking** (100% demand)
2. **Estimated arrival times** (100% demand)
3. **Delay notifications** (72.7% demand)
4. **Interactive route maps** (63.6% demand)
5. **Crowd level indicators** (45.5% demand)

### **Testing Results**
- **90% success rate** in completing navigation tasks
- **4.2/5.0 average satisfaction** score
- **Positive feedback** on real-time tracking features
- **Requests for** additional notification options
- **Appreciation for** accessibility considerations
